# EnergyChain 能源公链 — 架构与功能清单

## 一、概述

EnergyChain 是一条面向能源行业的应用型公链，基于 **Cosmos SDK + Cosmos EVM** 双引擎架构构建。链上同时支持 Cosmos 原生模块和 EVM 智能合约，具备 BFT 即时终结性、EIP-1559 费率市场以及完整的 EVM 兼容性（Solidity 合约、MetaMask、ethers.js 等生态工具可直接使用）。

核心定位：**为能源数据提供不可篡改的链上存证与溯源基础设施**。

## 六、模块架构详解

### 6.1 x/energy — 能源数据模块

```
用途：能源侧数据（发电量、用电量、电表读数等）的链上存证

数据结构：
  EnergyData {
    id         string   // 自增 ID（energy-1, energy-2...）
    category   string   // 用户自定义分类（如 "active_power", "meter_reading"）
    submitter  string   // 提交者地址
    data_hash  string   // 原始数据的 keccak256 哈希
    metadata   string   // 可选 JSON 元数据（电表号、时段、摘要等）
    block      int64    // 入块高度
    timestamp  int64    // 入块时间戳
  }

索引方式：
  · 按 ID 查询
  · 按 category 查询
  · 按 submitter 查询
  · 全量遍历

设计原则：只存哈希不存原始数据，保护隐私，链不定义数据结构。
```

### 6.2 x/oracle — 预言机模块

```
用途：外部数据源（电价、天气、碳价等）上链

数据结构：
  OracleData {
    category   string   // 数据分类（如 "electricity_price", "carbon_price"）
    value      string   // 数据值（字符串，灵活支持任意格式）
    metadata   string   // 可选元数据
    timestamp  int64
    submitter  string
    block      int64
  }

  OracleInfo {
    address              string
    name                 string
    active               bool
    authorized_categories []string  // 允许提交的分类列表
  }

安全机制：只有注册并授权的 Oracle 才能提交对应分类的数据。
```

### 6.3 x/identity — 身份管理模块

```
用途：能源行业参与者的链上身份注册与角色管理

数据结构：
  Identity {
    address       string   // 链上地址
    name          string   // 名称
    role          string   // 角色（如 "power_station", "grid_operator", "consumer"）
    status        string   // pending / active / revoked
    metadata      string   // 可选扩展信息
    registered_at int64
    updated_at    int64
  }

工作流：注册(pending) → 审批(active) → 可吊销(revoked)
```

### 6.4 x/audit — 审计日志模块

```
用途：记录链上关键操作的不可篡改审计轨迹

数据结构：
  AuditLog {
    id         string   // 自增 ID
    event_type string   // 事件类型（如 "data_submit", "identity_register"）
    actor      string   // 操作者地址
    target     string   // 操作对象（可选）
    action     string   // 操作描述
    data       string   // 附加数据 JSON（可选）
    block      int64
    timestamp  int64
    tx_hash    string   // 交易哈希（可选）
  }

查询维度：按操作者 / 按事件类型 / 按时间范围
```

## 八、安全特性

| 特性 | 说明 |
|------|------|
| BFT 共识 | 1/3 容错，即时终结性，无分叉 |
| EIP-1559 Gas | 动态费率防止垃圾交易 |
| 哈希防重复 | 合约层 `_hashToId` 映射，同一哈希只能存证一次 |
| 预言机授权 | Oracle 需注册并获授权才能提交特定分类数据 |
| 身份角色管理 | 链上地址与身份绑定，支持吊销 |
| 审计不可篡改 | 所有关键操作自动记录到 x/audit |
| 私钥隔离 | eth_secp256k1 密钥体系，兼容硬件钱包 |
