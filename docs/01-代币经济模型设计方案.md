# ECY 代币经济模型设计方案

## 一、代币基本信息

| 属性 | 值 |
|------|-----|
| 代币名称 | Energy Chain Yield |
| 代币符号 | ECY |
| 最小单位 | uecy (micro ECY) |
| 精度 | 18 位（1 ECY = 10^18 uecy） |
| 总供应量 | 1,000,000,000 ECY（10 亿） |
| Chain ID（主网） | energychain_1-1 |
| 共识机制 | CometBFT（基于 Cosmos SDK v0.54） |
| Gas 机制 | EIP-1559（Base Fee 动态调整 + Priority Fee） |

## 三、锁仓与释放计划（Vesting）

### 3.1 各类别释放时间表

| 类别 | 锁仓期 (Cliff) | 释放周期 | TGE 解锁比例 | 完全释放时间 |
|------|---------------|---------|-------------|------------|
| 质押奖励 | 无 | 链上通胀机制自动释放 | 0% | 约 10 年 |
| 生态基金 | 无 | 按需审批释放 | 5% | 5 年 |
| 团队顾问 | 12 个月 | 36 个月线性释放 | 0% | 4 年 |
| 能源激励 | 无 | 按季度释放 | 5% | 5 年 |
| 私募轮 | 6 个月 | 18 个月线性释放 | 0% | 2 年 |
| CEX 流动性 | 无 | TGE 全部可用 | 100% | 立即 |
| 公募/IDO | 无 | TGE 全部可用 | 100% | 立即 |
| 社区空投 | 无 | 分 4 次空投 | 25% | 1 年 |

### 3.2 初始流通量计算

TGE（Token Generation Event）时的初始流通量：

```

```

### 3.3 技术实现方式

Cosmos SDK 内置 `x/vesting` 模块支持链上锁仓，无需智能合约：

**连续线性释放账户（Continuous Vesting Account）**
- 适用于：团队、私募
- 创建方式：`energychaind tx vesting create-vesting-account <address> <amount> <end_time> --delayed`
- 链级别强制，代币在锁仓期内无法转账

**周期性释放账户（Periodic Vesting Account）**
- 适用于：生态基金、能源激励
- 支持自定义每期释放量和间隔
- 创建方式：`energychaind tx vesting create-periodic-vesting-account <address> <start_time> <periods_json>`

**延迟释放账户（Delayed Vesting Account）**
- 适用于：Cliff 期间完全锁定的场景
- 到期一次性全额释放

## 四、通胀与销毁机制

### 4.1 通胀模型（x/mint 模块）

| 参数 | 值 | 说明 |
|------|-----|------|
| 初始通胀率 | 8% | 年化 |
| 最大通胀率 | 15% | 质押率低于目标时通胀率上升 |
| 最小通胀率 | 3% | 质押率高于目标时通胀率下降 |
| 目标质押率 | 67% | 当全网质押比例 = 67% 时通胀率稳定 |
| 通胀率变化速率 | 每年最多变化 7% | 通过 `RateChange` 参数控制 |

**通胀调节逻辑：**
- 质押率 < 67% → 通胀率逐步上升（最高 15%），激励更多人质押
- 质押率 > 67% → 通胀率逐步下降（最低 3%），减少稀释
- 质押率 = 67% → 通胀率稳定在当前值

**年产出预估（以初始 8% 通胀率计算）：**
- 第 1 年：~80,000,000 ECY 增发
- 通胀分配：验证人佣金 + 委托人质押收益 + 社区税

### 4.2 销毁机制

**EIP-1559 Base Fee 销毁**
- 每笔 EVM 交易的 Base Fee 部分直接销毁（从流通中移除）
- 当前 Base Fee：10 Gwei（10,000,000,000 uecy）
- 交易越多，销毁越多，形成通缩压力

**治理销毁**
- 治理提案中的保证金（deposit），若提案未通过或被否决，保证金将被销毁
- 最小保证金：10,000 ECY

**Slashing 销毁**
- 验证人双签（Double Sign）：质押 5% 销毁
- 验证人宕机（Downtime）：质押 0.01% 销毁

### 4.3 长期供需模型

```
年份    总供应量(亿)    通胀增发(亿)    预估销毁(亿)    净增发(亿)
─────────────────────────────────────────────────────────────
  1      10.80          0.80           0.05           0.75
  2      11.53          0.73           0.08           0.65
  3      12.15          0.62           0.12           0.50
  4      12.60          0.45           0.15           0.30
  5      12.85          0.35           0.20           0.15
  ...    趋于稳定       递减            递增            趋于 0
```

随着生态交易量增长（RWA、DEX），EIP-1559 销毁量可能超过通胀增发，实现通缩。

## 五、代币使用场景

### 5.1 Gas 支付
- 所有链上交易（转账、合约调用、治理投票）必须用 ECY 支付 Gas
- EIP-1559 机制：Base Fee（自动调整，被销毁）+ Priority Fee（给验证人）

### 5.2 质押与安全
- 验证人质押 ECY 参与共识出块
- 委托人质押 ECY 到验证人，获得质押奖励
- 质押最小门槛：验证人自质押 ≥ 1,000 ECY

### 5.3 治理
- 持有并质押 ECY 的地址可参与链上治理投票
- 治理范围：参数修改、模块升级、社区基金使用

### 5.4 RWA 生态
- ECY 作为 RWA Token 的交易对基础货币（如 RWA/ECY 流动性池）
- 能源数据上链存证需消耗 ECY（Gas）
- 收益分配可以 ECY 或稳定币形式发放

### 5.5 跨链
- 通过 IBC 协议将 ECY 转移到其他 Cosmos 生态链
- 未来可通过桥接协议转移到以太坊等外部链

## 七、CEX 上币相关的 Tokenomics 要求

### 7.1 交易所需要确认的数据

| 数据项 | 值 |
|--------|-----|
| 总供应量 | 1,000,000,000 ECY |
| 初始流通量 | 130,000,000 ECY（13%） |
| 通胀模型 | 动态通胀 3%-15%，目标质押率 67% |
| 是否有销毁 | 是，EIP-1559 Base Fee 销毁 |
| 合约审计 | 需要（链代码 + 智能合约双审计） |
| Token 类型 | 原生 Gas/Staking 代币（非 ERC-20） |
| 充提方式 | 标准以太坊 JSON-RPC |
| 区块确认数 | 1（CometBFT 即时终局性） |

### 7.2 CoinGecko / CoinMarketCap 上架信息

- Token 名称、符号、Logo
- 总供应量、流通量、最大供应量
- 合约地址（原生代币无合约地址，提供链 RPC 端点）
- 区块浏览器链接（Blockscout）
- 项目官网、白皮书、社交媒体链接
- 交易所交易对信息
