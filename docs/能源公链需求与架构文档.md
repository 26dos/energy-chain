# 能源公链需求与架构文档（中国市场版）

> 面向中国电力市场与能源数据上链场景，整理公链需求、系统架构与功能点。

---

## 一、需求概述

### 1.1 业务背景

- **中国电力市场化**：2024年市场化交易电量约 6.2 万亿千瓦时，占全社会用电量超 60%，用户侧、售电侧、虚拟电厂等主体参与度持续提升。
- **多层次市场**：省间/区域/省内市场协同，覆盖年度/月度/月内/日前/日内交易，中长期、现货、辅助服务、绿电绿证等品种并存。
- **数据可信与共享**：合同、交易、计量、结算、充电桩运营等数据需要可追溯、防篡改、可审计，并支持跨主体、跨平台共享与合规披露。

### 1.2 公链定位

- **底层能力**：与以太坊类似——支持智能合约、去中心化共识、账户与资产模型、可扩展的链上存储与跨链能力。
- **能源特色**：面向能源行业设计账户模型、数据上链格式、合规与隐私方案，支撑用电侧、充电侧及各类能源数据上链与业务应用。

### 1.3 目标用户与参与方

| 参与方 | 典型场景 |
|--------|----------|
| 电力用户（工商业/居民） | 零售套餐签约、用电数据上链、绿电消费证明 |
| 售电公司 | 零售合同、套餐管理、结算与对账 |
| 虚拟电厂（聚合商） | 聚合关系、调节合同、现货/辅助服务收益分配 |
| 充电桩运营商 | 充电订单、支付、负荷与运行数据上链 |
| 新能源/储能/分布式资源 | 绿电交易、中长期/现货报价、计量与结算 |
| 电网/交易机构 | 市场规则上链、清算数据存证、监管审计 |
| 第三方应用/数据方 | 能源数据交易、碳/绿证关联、数据分析服务 |

## 二、公链核心功能（类 ETH 能力）

### 2.1 链层基础

| 功能模块 | 说明 | 与中国市场关联 |
|----------|------|----------------|
| 共识机制 | PoS / 兼容 EVM 的 BFT 等，保证去中心化与性能 | 满足高 TPS、低延迟，支撑交易与结算频率 |
| 账户模型 | EOA + 合约账户，支持多签与权限委托 | 企业多角色、代理交易、售电/虚拟电厂主体身份 |
| 原生代币 | Gas 费、质押、治理 | 可设计能源相关治理与激励（如绿电激励） |
| 智能合约 | 图灵完备、EVM 兼容或自有 VM | 合同、套餐、结算规则、拍卖逻辑上链执行 |
| 存储与状态 | 状态树、事件日志、可扩展的 calldata/storage | 合同存证、计量数据哈希、审计轨迹 |

### 2.2 扩展与互操作

- **Layer2 / 侧链**：高并发交易、批量结算、隐私计算可放在 L2 或专用侧链，主链做清算与存证。
- **跨链**：与其它能源链、联盟链、身份/支付链的资产与消息互通，支撑跨省、跨市场数据与结算。
- **预言机**：电力现货价格、负荷预测、气象、碳价等链下数据安全上链，供合约使用。

### 2.3 合规与隐私

- **身份与准入**：支持 KYC/企业认证、白名单、监管节点，满足国内对能源交易主体的合规要求。
- **隐私**：敏感用电/充电数据可仅上链哈希或加密密文，明文经授权解密；零知识证明用于“证明合规但不泄露明细”。
- **审计与监管**：链上审计接口、监管节点读权、数据导出与司法存证对接。

## 三、能源业务功能点

### 3.1 用电侧数据上链

#### 3.1.1 虚拟电厂相关

| 功能点 | 描述 | 上链内容建议 |
|--------|------|--------------|
| 聚合关系绑定 | 用户/负荷与虚拟电厂运营商的聚合关系 | 主体 ID、签约时间、容量/可调节能力、合约哈希 |
| 虚拟电厂合同 | 参与调节的协议与分成规则 | 合同版本、双方身份、调节类型、收益分配规则（可指向合约） |
| 现货价格联动零售套餐 | 与日前/实时价格联动的零售套餐（参考山东等试点） | 套餐 ID、价格公式、生效周期、用户绑定关系 |
| 调节事件与结算 | 实际响应量、价格、收益 | 事件 ID、时间、调节量、价格、收益分配结果、结算状态 |

**参考政策**：国家发改委、能源局《电力现货市场基本规则》；山东等省虚拟电厂参与现货与零售套餐规则。

#### 3.1.2 零售市场

| 功能点 | 描述 | 上链内容建议 |
|--------|------|--------------|
| 零售套餐 | 售电公司与用户签订的零售套餐（固定价、联动价等） | 套餐类型、价格结构、期限、售电公司/用户、合同哈希 |
| 零售合同 | 具体用户的购电合同 | 合同 ID、双方、电量、价格、违约与争议条款哈希 |
| 套餐与现货衔接 | 零售套餐与批发/现货市场的衔接关系 | 套餐与市场品种的映射、结算规则引用 |

#### 3.1.3 中长期交易

| 功能点 | 描述 | 上链内容建议 |
|--------|------|--------------|
| 双边协商/集中交易 | 年度、月度、月内等中长期合约 | 交易 ID、买卖方、电量、价格、交割周期、合约哈希 |
| 合同转让/回购 | 中长期合同流转 | 转让记录、新持有方、变更时间 |
| 与现货衔接 | 中长期与日前/实时的衔接与偏差处理 | 衔接规则引用、偏差结算参数存证 |

#### 3.1.4 现货交易

| 功能点 | 描述 | 上链内容建议 |
|--------|------|--------------|
| 日前/日内/实时出清结果 | 节点价格、出清电量等（可脱敏或汇总） | 市场周期、区域/节点、价格与电量哈希或汇总统计 |
| 报量报价与成交 | 主体报价与成交结果存证 | 主体匿名/脱敏 ID、时段、量价、成交结果哈希 |
| 偏差与考核 | 实际与计划偏差、考核结果 | 主体、时段、偏差量、考核结果、结算金额哈希 |

**说明**：现货敏感数据可按“哈希 + 授权解密”或“汇总统计上链、明细链下”方式处理，兼顾合规与隐私。

### 3.2 充电桩数据上链

| 功能点 | 描述 | 上链内容建议 |
|--------|------|--------------|
| 充电订单 | 单次充电的起止时间、电量、金额 | 订单 ID、桩 ID、用户脱敏 ID、电量、金额、时间、状态 |
| 支付与分账 | 用户支付、平台/场站/运营商分账 | 支付流水 ID、金额、分账比例与结果、结算状态（可引用链下支付系统） |
| 充电桩负荷与运行 | 功率曲线、利用率、故障与状态 | 桩 ID、时段、负荷/电量汇总或哈希、状态事件 |
| 充电桩资产与归属 | 桩的归属、运维主体、接入容量 | 资产 ID、运营商、容量、接入时间、合约哈希 |

**参考**：上海等地“区块链+充电桩”分账与交易存证实践，分账误差率可控制在极低水平，实现“交易即结算”式存证。

### 3.3 其他能源数据上链

| 数据类型 | 说明 | 上链方式建议 |
|----------|------|--------------|
| 绿电/绿证交易 | 绿电合同、绿证划转、消费证明 | 交易 ID、电量、证书编号、买卖方、时间、合约状态 |
| 分布式光伏/储能 | 并网协议、发电/放电量、参与市场记录 | 资源 ID、计量周期、电量哈希、市场参与记录 |
| 计量与抄表 | 电表读数、计量点数据 | 计量点 ID、周期、读数或哈希、抄表时间（可按需批量上链） |
| 碳/能效数据 | 碳排放、能效指标、碳配额 | 主体、周期、排放量/能效值或哈希、证书或配额引用 |
| 辅助服务 | 调频、调峰等申报与结算 | 服务类型、主体、时段、量、价格、结算结果哈希 |

## 四、系统架构建议

### 4.1 分层架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    应用层 (DApps / 业务系统)                      │
│  售电平台 | 虚拟电厂平台 | 充电运营 | 交易中心 | 数据服务 | 监管端  │
├─────────────────────────────────────────────────────────────────┤
│                    合约层 (智能合约 / 标准协议)                   │
│  零售合同 | 聚合关系 | 充电订单 | 交易存证 | 结算规则 | 访问控制   │
├─────────────────────────────────────────────────────────────────┤
│                    公链核心层 (类 ETH)                            │
│  共识 | 执行 (EVM/VM) | 存储 | 账户 | Gas | 跨链/预言机           │
├─────────────────────────────────────────────────────────────────┤
│                    网络与节点层                                  │
│  共识节点 | RPC/API | 监管节点 | 身份与准入                        │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 数据流与存证策略

- **高敏感（如用户明细、实时位置）**：仅上链哈希或承诺，明文存链下或授权解密服务。
- **业务关键（合同、成交、结算）**：关键字段上链存证，支持审计与争议调处。
- **公开或脱敏（市场出清价、汇总负荷）**：可直接上链或通过预言机写入，供合约与数据分析使用。

### 4.3 身份与权限

- **链上身份**：与实体身份/企业统一社会信用代码映射（可经合规 KYC 服务绑定）。
- **角色**：用户、售电公司、虚拟电厂、充电运营商、电网/交易机构、监管等，不同角色可访问不同合约与数据范围。
- **多签与代理**：企业多签钱包、交易员代理操作、结算自动执行等，通过合约与权限模型实现。

### 4.4 与现有系统对接

- **交易平台**：与各省电力交易中心、虚拟电厂/负荷聚合平台对接，将合同、申报、出清、结算结果按规则上链。
- **充电平台**：与充电运营商/聚合平台对接，订单、支付、负荷数据标准化后上链。
- **计量与采集**：与采集系统、电表/物联设备对接，计量数据哈希或周期汇总上链。

## 五、合规与监管考量（中国市场）

### 5.1 政策与标准

- **电力市场**：遵循《电力现货市场基本规则》及各省现货、零售、虚拟电厂等规则，上链数据与业务逻辑需与现行市场规则一致。
- **数据与安全**：《数据安全法》《个人信息保护法》——用户级数据需脱敏或加密，跨境与敏感数据需满足分类分级要求。
- **区块链**：关注网信办区块链备案、等保等要求；若涉及数字资产，需考虑金融监管边界。

### 5.2 建议

- 与电网、交易中心、能源主管部门保持沟通，链上数据模型与接口设计预留监管审计与上报能力。
- 优先在试点省份（如山东、广东、山西等现货与虚拟电厂较成熟地区）做业务闭环，再推广。
- 隐私方案（哈希、加密、零知识证明）与访问控制结合，实现“可用可审计、敏感不裸奔”。

## 七、附录：名词与参考

- **虚拟电厂**：聚合分布式负荷、储能、分布式电源等，参与电力市场与辅助服务。
- **零售套餐**：售电公司向用户提供的电费套餐，如固定价、峰谷、与现货联动等。
- **中长期交易**：年度、月度、月内等较长期限的电力合约交易。
- **现货交易**：日前、日内、实时等短期电力市场。
- **参考**：《电力现货市场基本规则》；山东虚拟电厂现货价格联动零售套餐；上海区块链充电桩分账案例；多链式能源区块链电力交易模型等。
